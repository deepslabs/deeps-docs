## Node Configuration

### Prerequisites

Below is a list of standard requirements for running nodes on the SafeX chain:

1. **Operating system:** Ubuntu 18.04 or Linux Kernel 5.16.
2. **Hardware:**
   * CPU: 4 Cores
   * Memory: 16GB RAM
   * Storage: 1TB SSD/HDD
   * Network: At least 20Mb bandwidth with an independent IP address

## Node Initialization

### Download Binaries

<Callout type="warn">
Compiling from binaries is temporarily unsupported since the network is not open-source at the Testnet stage.
</Callout>

### Using Docker

**Docker** is the only available way to operate nodes on SafeX chain at the alpha testnet stage. Developers can quickly run nodes through the images posted by SafeX team via the following command:

```shell
docker run -p 30333:30333 boolnetwork/bnk-node:alpha-testnet --validator --chain=alpha_testnet --name <name on telemetry>
```

If you prefer to run in the background, please add `-d` to the docker command.

If you prefer to persist the node's data on the host machine, please add `-v ./data:/data` to the docker command.

If you intend to run an RPC service that is externally accessible for applications or third-party applications, such as Polkadot.js, please add the following flags to run a full node:

* `--unsafe-rpc-external`: Listen to all RPC interfaces.
* `--unsafe-ws-external`: Listen to all Websocket interfaces.

More details can be found by executing the following command:

```shell
docker run --rm boolnetwork/bnk-node:alpha-testnet --help
```

## FAQ

#### Why are nodes out of sync?

If the number of peers on your node is unstable and occasionally reaches 0, causing normal synchronization, try adding port mapping on your router to your public IP: 30333 → node's internal IP: 30333, and set the firewall to allow port 30333 to connect.
